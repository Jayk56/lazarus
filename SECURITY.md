# Security policy

## Supported versions

Security fixes are released for the newest Lazarus release. A different support
window must be agreed and documented by the organization operating the service.

## Report a vulnerability

Use GitHub's private vulnerability reporting for this repository. Do not open a
public issue for an unpatched vulnerability or include secrets, production data,
host names, or customer details in a report.

Include the affected version or image digest, expected and observed behavior,
reproduction steps, and likely impact. Maintainers will coordinate disclosure
and publish a GitHub Security Advisory when a release fixes a confirmed issue.

This community project does not promise a response-time or remediation-time
service level. Organizations that require contractual support should establish
that support separately before production use.
